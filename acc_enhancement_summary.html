Agent Communication Center Enhancement

Upgrading to a fully-featured messaging system for BoarderframeOS

📡 Current ACC

  • Limited to 5 hardcoded agents
  • No message persistence
  • WebSocket disabled
  • No message bus integration
  • Single global chat room
  • No agent presence tracking
  • Basic Claude integration only
  • No threading support

🚀 Enhanced ACC

  • Dynamic agent discovery
  • PostgreSQL message storage
  • Real-time WebSocket updates
  • Full message bus integration
  • Channel-based messaging
  • Agent presence & status
  • Agent-to-agent messaging
  • Thread conversations

🔄 Migration Steps

1
Run Migration Script
python scripts/migrate_to_enhanced_acc.py
Creates database schema and default channels
2
Test Enhanced ACC
python test_acc_enhanced.py
Verifies database connection and features
3
Update Startup Configuration
python scripts/update_startup_for_enhanced_acc.py
Updates startup.py to use enhanced version
4
Restart BoarderframeOS
python startup.py
Enhanced ACC will start automatically

🌟 Key Benefits

💾

Persistent History

All messages stored in PostgreSQL for full conversation history

📢

Organized Channels

Department, topic, and broadcast channels for structured communication

🤖

Agent Autonomy

Agents can message each other directly through the message bus

🟢

Presence Tracking

See which agents are online, busy, or away in real-time

🧵

Threaded Conversations

Keep related messages together with thread support

⚡

Real-time Updates

WebSocket connections for instant message delivery

⚠️
Important: The enhanced ACC requires PostgreSQL to be running on port 5434. Make sure to start it with docker-compose up -d postgresql before launching.
