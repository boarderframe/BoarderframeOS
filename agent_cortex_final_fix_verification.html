❌ The Problem

Agent Cortex was failing to start due to environment pollution from the parent process. Flask/Werkzeug environment variables like WERKZEUG_SERVER_FD were being inherited, causing startup failures.

KeyError: 'WERKZEUG_SERVER_FD'

Even with multiple attempts to clean the environment, the subprocess was still inheriting problematic variables from the startup.py process.

✅ The Solution

Created an isolated launcher that creates a completely clean environment with only the essential variables needed for Agent Cortex to run.

  • ✓ Clean environment with minimal variables
  • ✓ No inheritance of Flask/Werkzeug settings
  • ✓ Explicit PYTHONPATH configuration
  • ✓ Inline Python execution for reliability

🔧 Implementation Details

Key Components:

# Clean environment creation clean_env = { 'PATH': os.environ.get('PATH', ''), 'HOME': os.environ.get('HOME', ''), 'USER': os.environ.get('USER', ''), 'PYTHONPATH': project_root, 'FLASK_DEBUG': '0', 'LANG': os.environ.get('LANG', 'en_US.UTF-8'), 'LC_ALL': os.environ.get('LC_ALL', 'en_US.UTF-8'), } # Inline execution to avoid import issues code = ''' import sys sys.path.insert(0, "{project_root}") from ui.agent_cortex_management import AgentCortexManagementUI ui = AgentCortexManagementUI() ui.app.run(host="0.0.0.0", port=ui.port, debug=False, use_reloader=False) ''' # Run with clean environment subprocess.run([python_exe, '-c', code], env=clean_env, cwd=project_root)
