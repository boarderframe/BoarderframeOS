🚀 Quick Start Commands

# Start Redis (required) docker-compose up -d redis # Start workers manually ./manage_workers.py cluster --workers 2 # Or start everything via startup.py (includes workers) python startup.py # Monitor workers ./manage_workers.py monitor # Submit test task ./manage_workers.py submit "core.task_queue.agent_think" '{"agent_id": "test"}' --priority high # View dashboard open task_queue_dashboard.html

⚠️ Important Notes

  • • Redis must be running before starting workers
  • • Celery is already in requirements.txt (version 5.3.4)
  • • Workers will auto-start when using startup.py
  • • Check logs in logs/celery_worker_*.log for issues
