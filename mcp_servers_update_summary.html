  • Business Services (3): Analytics, Payment, Customer

Health Check Process

  1. HTTP request to /health endpoint
  2. Timeout: 2 seconds per server
  3. Status: healthy, error, timeout, offline
  4. Updates system_status.json
  5. Notifies metrics layer
  6. Triggers UI refresh

API Endpoints

  • GET /servers - Servers monitoring page
  • POST /api/servers/refresh - Refresh server status
  • GET /health - Individual server health
  • POST /api/global/refresh - Global system refresh
